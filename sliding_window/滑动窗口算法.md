# 滑动窗口算法

---

## 一、核心思想

滑动窗口算法是一种用于解决**数组/字符串**中**子区间问题**的高效技巧，其核心思想是：

1. **维护一个窗口**：用两个指针（通常称为 `left` 和 `right`）表示窗口的左右边界。
2. **窗口扩张**：`right` 指针不断向右移动，扩大窗口，将新元素纳入窗口。
3. **窗口收缩**：当窗口内的状态不满足条件时，`left` 指针向右移动，缩小窗口，直到重新满足条件。
4. **更新答案**：在每次窗口调整后，根据题目要求更新最优解（最大值、最小值、计数等）。

通过这种方式，每个元素最多被访问两次（一次被 `right` 加入，一次被 `left` 移除），因此时间复杂度为 **O(n)**，相比暴力枚举所有子区间的 O(n²) 有显著提升。

## 二、适用场景

- 求满足某种条件的**最长/最短**子数组
- 求满足某种条件的子数组**个数**
- 求满足某种条件的子数组的**最大/最小和**

## 三、模板代码（C++）

以经典问题 **"无重复字符的最长子串"** 为例：

> 给定一个字符串 `s`，找出其中不含重复字符的最长子串的长度。

```cpp
int lengthOfLongestSubstring(string s) {
    unordered_map<char, int> window; // 记录窗口内每个字符的出现次数
    int left = 0, right = 0;
    int maxLen = 0;

    while (right < s.size()) {
        // 1. 扩张窗口：将 s[right] 加入窗口
        char c = s[right];
        window[c]++;
        right++;

        // 2. 收缩窗口：当窗口内出现重复字符时，移动 left
        while (window[c] > 1) {
            char d = s[left];
            window[d]--;
            left++;
        }

        // 3. 更新答案：当前窗口 [left, right) 是一个合法窗口
        maxLen = max(maxLen, right - left);
    }

    return maxLen;
}
```

### 算法流程图解

以 `s = "abcabcbb"` 为例：

```
步骤    left    right    窗口内容      maxLen
 1       0       1       "a"            1
 2       0       2       "ab"           2
 3       0       3       "abc"          3
 4       0→1     4       "bca"          3    (遇到重复'a', left右移)
 5       1→2     5       "cab"          3    (遇到重复'b', left右移)
 6       2→3     6       "abc"          3    (遇到重复'c', left右移)
 7       3→4     7       "bcb"→"cb"     3    (遇到重复'b', left右移)
 8       4→5     8       "bb"→"b"       3    (遇到重复'b', left右移)
```

最终结果：**3**

### 复杂度分析

| 复杂度 | 值 | 说明 |
|--------|------|------|
| 时间复杂度 | O(n) | 每个元素最多被访问两次 |
| 空间复杂度 | O(min(n, m)) | m 为字符集大小，取决于哈希表的大小 |

## 四、题目实战

### 4.1 LeetCode 2090 - 半径为 k 的子数组平均值

#### 题目描述

给你一个由 `n` 个元素组成的整数数组 `nums` 和一个整数 `k`。

请你找出**平均数最大**且**长度为 `2k+1`** 的连续子数组，并输出该最大平均数。对于每个位置 `i`，以它为中心、左右各取 `k` 个元素组成子数组。如果某个位置左右不足 `k` 个元素，则该位置结果为 `-1`。

#### 解题代码（单循环滑动窗口）

```cpp
class Solution {
public:
    vector<int> getAverages(vector<int>& nums, int k) {
        int n = nums.size();
        int windowSize = 2 * k + 1;
        vector<int> ans(n, -1);  // 默认全为 -1
        long long sum = 0;

        for (int i = 0; i < n; i++) {
            sum += nums[i];  // 加入右边新元素

            if (i >= windowSize - 1) {  // 窗口已满
                int center = i - k;
                ans[center] = sum / windowSize;
                sum -= nums[center - k];  // 移除左边即将滑出的元素
            }
        }

        return ans;
    }
};
```

#### 💡 疑问：`i >= windowSize - 1` 是什么意思？

> `i` 不是索引吗？为什么和窗口大小比较？

`i` 确实是索引。这个条件的含义是：**当索引 `i` 走到 `windowSize - 1` 时，恰好从索引 0 到 `windowSize - 1` 收集了 `windowSize` 个元素，窗口第一次满了。**

以 `nums = [7,4,3,9,1,8,5,2,6]`, `k = 3`（`windowSize = 7`）为例：

```
i = 0: 已收集 [7]                     → 1个元素，窗口没满
i = 1: 已收集 [7,4]                   → 2个元素，窗口没满
i = 2: 已收集 [7,4,3]                 → 3个元素，窗口没满
i = 3: 已收集 [7,4,3,9]               → 4个元素，窗口没满
i = 4: 已收集 [7,4,3,9,1]             → 5个元素，窗口没满
i = 5: 已收集 [7,4,3,9,1,8]           → 6个元素，窗口没满
i = 6: 已收集 [7,4,3,9,1,8,5]         → 7个元素，窗口满了！(i == windowSize-1)
       窗口 [0..6]，中心 = i - k = 3
       ans[3] = (7+4+3+9+1+8+5) / 7 = 5
i = 7: 加入 nums[7]=2，减去 nums[0]=7  → 窗口 [1..7]，中心 = 4
       ans[4] = (4+3+9+1+8+5+2) / 7 = 4
i = 8: 加入 nums[8]=6，减去 nums[1]=4  → 窗口 [2..8]，中心 = 5
       ans[5] = (3+9+1+8+5+2+6) / 7 = 4
```

#### 💡 疑问：如何处理"左右不足 k 个元素"的情况？

**不需要显式判断**，而是通过 **初始化 + 只更新合法位置** 来隐式处理：

1. **初始化**：`vector<int> ans(n, -1)` — 所有位置默认值就是 `-1`
2. **只更新合法位置**：循环中只有当 `i >= windowSize - 1`（窗口已满）时，才写入 `ans[center]`
3. **效果**：
   - 前 k 个位置（0 到 k-1）：窗口还没满，永远不会被写入 → 保持 `-1`
   - 后 k 个位置（n-k 到 n-1）：窗口已经滑过，永远不会被写入 → 保持 `-1`
   - 中间位置：窗口满时被正确更新为平均值

> **技巧：用默认值 + 选择性更新来替代显式的边界判断。**

### 4.2 LeetCode 2379 - 得到 K 个黑块的最少涂色次数

#### 题目描述

给你一个长度为 `n` 的字符串 `blocks`，其中 `blocks[i]` 要么是 `'W'`（白色）要么是 `'B'`（黑色）。同时给你一个整数 `k`。

请你找出**连续** `k` 个黑色块的子串，并返回将其中所有白色块涂成黑色所需的**最少操作次数**。

#### 解题代码（滑动窗口）

```cpp
class Solution {
public:
    int minimumRecolors(string blocks, int k) {
        int count = 0;  // 窗口内 'W' 的数量
        int ans = INT_MAX;

        for (int i = 0; i < (int)blocks.size(); i++) {
            if (blocks[i] == 'W') count++;  // 加入右边新元素

            if (i >= k - 1) {  // 窗口已满
                ans = min(ans, count);
                if (blocks[i - k + 1] == 'W') count--;  // 移除左边元素
            }
        }

        return ans;
    }
};
```

#### ⚠️ 常见错误：窗口满的判断条件写错

> **错误代码：`if(i >= k)` 应改为 `if(i >= k - 1)`**

窗口大小为 `k`，当索引 `i = k-1` 时，已经收集了索引 `0` 到 `k-1` 共 `k` 个元素，窗口**第一次满了**。

以 `blocks = "WBW"`, `k = 2` 为例：

```
错误写法 (i >= k):
  i=0: count=1(W), i>=2? No
  i=1: count=1(B), i>=2? No        ← 漏掉了窗口 [0,1]，应该 ans=1
  i=2: count=2(W), i>=2? Yes → ans=2  ← 此时 count 包含了3个元素！
       然后移除 blocks[0]='W' → count=1
  返回 ans=2 ❌（正确答案是1）

正确写法 (i >= k-1):
  i=0: count=1(W), i>=1? No
  i=1: count=1(B), i>=1? Yes → ans=1  ← 窗口 [0,1]，正确！
       然后移除 blocks[0]='W' → count=0
  i=2: count=1(W), i>=1? Yes → ans=1
       然后移除 blocks[1]='B' → count=1
  返回 ans=1 ✅
```

| 写法 | 结果 |
|------|------|
| `i >= k` | ❌ 窗口大小为 k 时，第一次满应该是 `i = k-1` |
| `i >= k - 1` | ✅ 正确的窗口满判断条件 |

> **通用规律：** 对于大小为 `k` 的滑动窗口，窗口第一次满的条件永远是 `i >= k - 1`，因为索引从 0 开始，`0` 到 `k-1` 恰好是 `k` 个元素。

### 4.3 LeetCode 2841 - 几乎唯一子数组的最大和

#### 题目描述

给你一个整数数组 `nums` 和两个整数 `m` 和 `k`。

返回长度为 `k` 的几乎唯一子数组的**最大和**。如果子数组中至少有 `m` 个互不相同的元素，则称该子数组为"几乎唯一"的。

#### 解题代码（滑动窗口 + 哈希表）

```cpp
class Solution {
public:
    long long maxSum(vector<int>& nums, int m, int k) {
        long long sum = 0, ans = 0;
        unordered_map<int, int> freq;

        for (int i = 0; i < (int)nums.size(); i++) {
            sum += nums[i];
            freq[nums[i]]++;

            if (i >= k - 1) {  // 窗口已满
                if ((int)freq.size() >= m) {
                    ans = max(ans, sum);
                }
                int leftVal = nums[i - k + 1];
                sum -= leftVal;
                if (--freq[leftVal] == 0) {
                    freq.erase(leftVal);
                }
            }
        }

        return ans;
    }
};
```

> **说明：** 这道题的解法已经是**最优解**（O(n) 时间，O(k) 空间），滑动窗口 + 哈希表是标准做法。

### 4.4 LeetCode 3679 - 平衡库存的最少丢弃数

#### 题目描述

给你两个整数 `w` 和 `m`，以及一个整数数组 `arrivals`，其中 `arrivals[i]` 是第 `i+1` 天到达的物品类型。

物品按以下规则管理：
- 每个到达的物品可以**保留**或**丢弃**；物品只能在到达当天被丢弃。
- 对于每一天 `i`，考虑窗口 `[max(1, i-w+1), i]`（最近 `w` 天）：在该窗口内，每种物品类型最多出现 `m` 次。
- 如果保留第 `i` 天的物品会导致其类型在窗口内超过 `m` 次，则必须丢弃。

返回需要丢弃的**最少**物品数。

#### 解题思路

这是一个**变长窗口**问题（窗口大小固定为 `w`，但窗口起始位置随天数变化）。

关键难点：窗口滑出时，需要知道滑出的物品是否被保留过——只有保留的物品才需要从频率计数中移除。

**解决方案：用 `kept[]` 数组记录每个位置的物品是否被保留。**

#### 解题代码（滑动窗口 + 哈希表）

```cpp
class Solution {
public:
    int minArrivalsToDiscard(vector<int>& arrivals, int w, int m) {
        int n = arrivals.size();
        unordered_map<int, int> freq;   // 当前窗口内每种类型的保留数量
        vector<bool> kept(n, false);    // 记录每个位置的物品是否被保留
        int discards = 0;

        for (int i = 0; i < n; i++) {
            // 1. 窗口滑出：移除超出窗口范围的物品
            if (i >= w && kept[i - w]) {
                freq[arrivals[i - w]]--;
            }

            // 2. 判断当前物品是否可以保留
            if (freq[arrivals[i]] < m) {
                freq[arrivals[i]]++;
                kept[i] = true;
            } else {
                discards++;
            }
        }

        return discards;
    }
};
```

#### 💡 关键技巧：用 `kept[]` 数组区分"保留"和"丢弃"

> **为什么需要 `kept[]` 数组？**

窗口滑出时，需要将滑出的物品从频率计数中移除。但如果该物品在到达时被**丢弃**了，它从未被计入频率，此时不应该减少频率。

```
arrivals = [1, 2, 3, 3, 3, 4], w = 3, m = 2

i=0: 保留 1, freq={1:1}, kept=[T,_,_,_,_,_]
i=1: 保留 2, freq={1:1,2:1}, kept=[T,T,_,_,_,_]
i=2: 保留 3, freq={1:1,2:1,3:1}, kept=[T,T,T,_,_,_]
i=3: 保留 3, freq={2:1,3:2}, kept=[T,T,T,T,_,_]  (窗口滑出 1)
i=4: 丢弃 3, freq={2:1,3:2}, kept=[T,T,T,T,F,_]  (窗口滑出 2, 但 2 被保留过所以 freq[2]--)
     → freq={3:2}, 3 已达上限 m=2，丢弃！
i=5: 保留 4, freq={3:2,4:1}, kept=[T,T,T,T,F,T]  (窗口滑出 3, 3 被保留过所以 freq[3]--)
```

> **通用规律：** 当滑动窗口中需要区分"有效"和"无效"元素时，用一个额外的数组记录每个位置的状态，窗口滑出时根据状态决定是否更新计数。

### 4.5 LeetCode 1423 - 可获得的最大点数

#### 题目描述

几张卡牌排成一行，每张卡牌都有一个对应的点数。给你一个整数数组 `cardPoints` 和一个整数 `k`。

每次行动，你可以从行的**开头**或**末尾**取一张卡牌，共取 `k` 次。返回你能获得的最大点数。

#### 解题思路：反向思维

从两端取 `k` 张牌，等价于**保留中间 `n-k` 张连续的牌不取**。要使取出的点数最大，就要使保留的点数最小。

所以问题转化为：**找长度为 `n-k` 的连续子数组的最小和**，答案 = 总和 - 最小和。

#### 解题代码（滑动窗口）

```cpp
class Solution {
public:
    int maxScore(vector<int>& cardPoints, int k) {
        int n = cardPoints.size();
        int m = n - k;  // 需要保留的连续子数组长度

        // 边界：k == n 时取所有牌，直接返回总和
        if (m == 0) {
            int total = 0;
            for (int x : cardPoints) total += x;
            return total;
        }

        int sum = 0, total = 0, minSum = INT_MAX;

        for (int i = 0; i < n; i++) {
            sum += cardPoints[i];
            total += cardPoints[i];

            if (i >= m - 1) {  // 窗口已满
                minSum = min(minSum, sum);
                sum -= cardPoints[i - m + 1];  // 移除左边元素
            }
        }

        return total - minSum;
    }
};
```

#### ⚠️ 常见错误：边界条件 `m = 0` 的处理位置

当 `k = n`（取所有牌）时，`m = n - k = 0`，此时 `m - 1 = -1`，条件 `i >= m - 1` 即 `i >= -1` **永远为真**。

循环内会执行 `sum -= cardPoints[i - m + 1]` 即 `sum -= cardPoints[i + 1]`，当 `i = n-1` 时访问 `cardPoints[n]`，**数组越界**！

```cpp
// ❌ 错误写法：边界检查放在循环之后
for (int i = 0; i < n; i++) {
    // 当 m=0 时，i >= -1 永远为真，循环内越界访问 cardPoints[n]
}
if (m == 0) return p;  // 越界已经发生了！

// ✅ 正确写法：边界检查放在循环之前
if (m == 0) return total;  // 提前返回，避免进入循环
for (int i = 0; i < n; i++) {
    // ... 正常滑动窗口逻辑
}
```

| 写法 | 结果 |
|------|------|
| 边界检查放在循环之后 | ❌ 循环内已经越界访问，未定义行为 |
| 边界检查放在循环之前 | ✅ 提前返回，避免进入循环 |

> **通用规律：** 当滑动窗口的窗口大小可能为 0 时，一定要在循环**之前**处理这个边界情况，否则循环内的索引计算会越界。

## 五、错误总结与训练建议

### 5.1 已犯错误回顾

| 题目 | 错误类型 | 具体问题 |
|------|---------|---------|
| 2379 | 窗口满判断条件差 1 | `i >= k` 应为 `i >= k - 1` |
| 1423 | 边界条件处理位置错误 | `m == 0` 的检查放在循环之后，导致越界 |
| 2090 | 理解疑问（非 bug） | 对 `i >= windowSize - 1` 的含义不清晰 |
| 2841 | 无错误 ✅ | — |

**核心问题：对滑动窗口的"边界条件"缺乏系统性的检查习惯。**

### 5.2 训练建议

#### ① 牢记窗口满的判断公式

窗口大小为 `k` 时，窗口第一次满的条件**永远是 `i >= k - 1`**（索引从 0 开始，0 到 k-1 共 k 个元素）。

> **自测方法：** 写完代码后，用 `k = 1`（最小窗口）手动走一遍。如果 `k = 1` 时条件不成立，说明写错了。

#### ② 写代码前先检查"窗口大小是否可能为 0"

在滑动窗口题目中，窗口大小 = `n - k` 或 `2k + 1` 等表达式。**写循环之前**，先问自己：
- 窗口大小能不能等于 0？
- 如果能，循环内的索引计算会不会越界？

> **养成习惯：** 在写 `for` 循环之前，先把窗口大小为 0 的情况用 `if` 提前返回。

#### ③ 用极端用例自测

写完代码后，**必须**用以下极端用例手动走一遍：

| 极端用例 | 检查什么 |
|---------|---------|
| `k = 1` | 窗口最小，窗口满条件是否正确 |
| `k = n` | 窗口等于整个数组，是否越界 |
| `n = 1` | 数组只有一个元素 |
| 全相同元素 | 边界是否正确处理 |

#### ④ 滑动窗口检查清单

每次写完滑动窗口代码，按以下清单逐项检查：

- [ ] 窗口满的条件是 `i >= k - 1`（不是 `i >= k`）
- [ ] 窗口大小为 0 时，是否在循环之前提前返回
- [ ] 移除左边元素时，索引是 `i - k + 1`（不是 `i - k`）
- [ ] 循环内访问的数组下标是否在 `[0, n-1]` 范围内
- [ ] 用 `k=1` 和 `k=n` 各走一遍
